# c3kit-wire 3.0.0 Release Notes

## Breaking Changes

### React 18 + Reagent 2

This is a **major version bump** because of the React 17 → 18 and Reagent 1.3 → 2.0 upgrade. All test helpers have been rewritten for React 18 compatibility.

### Functional Compiler by Default

Wire sets Reagent's **functional compiler** (`{:function-components true}`) for tests. All components render as React functional components under the hood. This ensures `with-let finally` blocks fire via `useEffect` cleanup on unmount — critical for proper test isolation.

**Impact:** Components that relied on class-specific lifecycle methods (`component-did-mount`, `component-did-update`) may need adjustment. Use `with-let` + `after-render` or `use-effect` (via `:f>`) instead.

### Synchronous Cleanup

Reagent's functional compiler defers `with-let finally` cleanup via microtasks (`Promise.resolve().then(...)`). In synchronous test execution, these microtasks never run. Wire overrides `reagent.impl.component/queue-cleanup` to run synchronously, ensuring:

- `with-let finally` blocks fire immediately on unmount
- Event listeners registered in `with-let` are properly removed
- Reagent reactions are disposed between tests

### Removed: `ReactTestUtils.Simulate`

All event simulation now uses **native DOM events** instead of `ReactTestUtils.Simulate`. The `cljsjs.react.dom.test-utils` dependency has been removed.

**Removed functions:**
- `simulate` / `simulate!` — replaced by specific event functions
- `simulator` — the `ReactTestUtils.Simulate` reference
- `stub-reset-swap` — no longer needed; `wire/reset!` and `wire/swap!` handle flushing directly

### Removed: `reagent.dom/render`

Rendering uses `reagent.dom.client/create-root` + `.render` (React 18 createRoot API). The `reagent.dom` namespace is no longer required.

### Changed: `render`

- Now manages React 18 roots internally (one root per container)
- Uses `flushSync` for synchronous rendering + single `act(noop)` to drain effects
- No longer sets up `js/Slider` or `js/jwplayer` stubs (those were app-specific)

### Changed: `flush`

Uses `flushSync(reagent/flush)` + `batch/flush-after-render` + single `act(noop)` to drain effects. This replaces the previous double-`act()` pattern for better performance.

### Changed: `reset!` / `swap!`

Performs the atom operation then calls `sync-flush`. No longer wrapped in `act()` directly — the flush handles all synchronization.

### Changed: `dispatch-event` (internal)

Event dispatch is no longer wrapped in `act()`. The native event fires synchronously, and the subsequent `flush` (called by `!` variants like `click!`, `change!`) handles rendering.

### Changed: `change` / `change!`

Element-type-aware dispatching:
- **Text inputs**: sets value via native setter (bypasses React's value tracker), dispatches `input` event
- **Select elements**: sets value via native setter, dispatches `change` event
- **Checkboxes/radios**: dispatches `click` event (only if state differs)
- **File inputs**: sets native `files` property, dispatches `change` event

### Changed: `check-box` / `check-box!`

Now dispatches a `click` event (only when checked state differs) instead of directly setting `.checked` and firing a synthetic change.

### Changed: `blur` / `blur!`

Now dispatches both `focusout` (bubbles) and `blur` (doesn't bubble). React 18 delegates `onBlur` via `focusout`, so dispatching only `blur` no longer triggers React's `onBlur` handler.

### Changed: `focus` / `focus!`

Now dispatches both `focusin` (bubbles) and `focus` (doesn't bubble), matching React 18's delegation model.

### Changed: `mouse-down` / `mouse-down!`

The second argument is now an opts map (or number for backwards compat) instead of just a button number. Defaults to `{:button 0 :buttons 1}`.

### Changed: `mouse-enter` / `mouse-leave`

Now dispatch both `mouseover`/`mouseenter` (or `mouseout`/`mouseleave`) to match how browsers fire these events, which React 18 relies on.

### Changed: `render-roots` internals

Changed from a vector of DOM nodes to a map of `{container -> react-root}`, supporting proper per-container root management and unmounting.

### Changed: `reset-dom!` / `unmount-render-roots`

Between-test cleanup now:
1. Unmounts roots via `flushSync` (triggers `useEffect` cleanup)
2. Flushes Reagent and after-render queues
3. Drains effects via `act(noop)`
4. Removes all tracked document/window event listeners

This prevents listener and reaction accumulation across tests that caused progressive slowdown.

## New Functions

### `c3kit.wire.spec-helper`

- **`act [f]`** — wraps `React.act()`, scoping `IS_REACT_ACT_ENVIRONMENT` to only be true during the call
- **`unmount` / `unmount [container]`** — unmounts a React 18 root (replaces `reagent.dom/unmount-component-at-node`)
- **`reset! [atom val]`** — `reset!` + synchronous flush. Use for Reagent atoms in tests.
- **`swap! [atom f & args]`** — `swap!` + synchronous flush. Same purpose as `reset!`.
- **`suppress-history-push-state!`** — suppresses `SecurityError` from `pushState`/`replaceState` in `file://` test environments. Call once in your spec_helper.
- **`focus-out` / `focus-out!`** — dispatches `focusout` only (without `blur`). Useful when you specifically need the bubbling focus event.
- **`scroll` / `scroll!`** — dispatches `scroll` event on an element
- **`drag-end` / `drag-end!`** — was missing from the previous version (drag-start existed but drag-end did not)

### All event functions now accept an opts map

Every event function (`click`, `key-down`, `mouse-move`, `touch-start`, `drag`, etc.) now has an additional arity accepting an opts map that gets merged into the event constructor. This allows passing event properties like `{:shiftKey true}`, `{:clientX 100}`, etc.

## Other Changes

- **Document/window listener tracking**: Wire intercepts `addEventListener`/`removeEventListener` on `document` and `window` to track all listeners. On `reset-dom!`, any remaining tracked listeners are forcefully removed. This is a safety net — properly written `with-let finally` blocks should remove their own listeners, but the tracking prevents leaks from any missed cleanup.
- **`google/oauth-button`**: rewritten from `create-class` (form-3) to `with-let` + `after-render`, eliminating `component-did-mount` lifecycle
- **`google/with-ref`**: simplified ref composition logic (no longer uses `juxt`)
- **`google/on-button-mount`**: event listener now uses `:once true`
- **`IS_REACT_ACT_ENVIRONMENT`**: set to `false` globally, enabled only inside `act()` calls — prevents noisy warnings from state updates in `before`/`after` blocks
- Code organized into IntelliJ-collapsible `;region` / `;endregion` sections
